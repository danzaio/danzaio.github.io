Operating loop

Prompt first. Engineer context. Orchestrate agents. Prove the loop.

Dan treats LLMs as operating surfaces. The workflow must show its scoped contract, usable context, coordinated handoffs when useful, validation path, and closeout evidence.

01

Turn the vague goal into a contract

Dan turns the target, privacy boundary, acceptance criteria, done state, and stop conditions into a contract before agents act.

02

Read current state before acting

The loop starts with docs, source, current state, prior decisions, examples, and edge cases, then keeps only the context needed for the task.

03

Split only when the work can split

Agents get bounded responsibilities, acceptance criteria, and one accountable merge into the final implementation.

04

Verify the actual surface

Closeout waits for tool-backed checks, real surface review, fixes from observed behavior, evidence, risks, and next actions.

Closeout checklist

Contracts checked: the result still matches the user goal, privacy boundary, acceptance criteria, and stop conditions.
Context checked: claims stay grounded in sources, observed state, or explicit inference.
Behavior checked: important paths are exercised through the right tools before handoff.
Issues fixed: layout drift, unreadable contrast, clipped controls, broken links, stale copy, and ungrounded claims.
Closeout names: evidence observed, unresolved risks, and the next useful action.
