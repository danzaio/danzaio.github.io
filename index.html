Method

I keep the process boring on purpose: scope the job, ground the context, build the smallest reliable system, then verify the surface users touch.

Method and proof

Orchestrate when the work actually splits. Keep one owner for the result.

01

Start from the user and the job

I name the person, workflow, tool surface, data boundary, and real done state before choosing an agent pattern.

02

Ground the system in current state

Docs, source, product behavior, examples, edge cases, and prior decisions shape the context the model receives.

03

Orchestrate only where it helps

Single-agent paths stay single. Multi-agent workflows get bounded responsibilities, acceptance criteria, and one merge owner.

04

Verify against the real surface

Closeout waits for tool-backed checks, browser review when needed, fixes from observed behavior, and clear handoff notes.
