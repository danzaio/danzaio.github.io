Six build types

Work organized by system shape.

Each build type starts with the product or operational outcome, then makes stack, review state, boundary, and proof visible enough for another person to trust.

AI workflow automation

Reliable agent workflow system

A system for turning broad AI work into scoped tasks, clear ownership, approval points, inspectable outputs, and a final handoff.

Teams can use agents for real work without losing context, reviewability, or accountability.

Stack: Next.js / TypeScript / agent routing / structured artifacts

Checks: routes work by risk / keeps one merge owner / returns proof instead of chat

Web task automation

Browser automation operator

A browser workflow for navigating real web apps, reading page state, filling forms, collecting evidence, and stopping before risky external effects.

Repetitive browser work becomes faster while sensitive actions stay visible to the operator.

Stack: Playwright / DOM state / form automation / approval gates

Checks: observes before acting / records page evidence / asks before submit

Source-grounded research

RAG research agent

A retrieval-first research flow that gathers sources, ranks useful context, extracts structured facts, and produces a reusable brief.

Research becomes a checked work product instead of a long transcript someone has to clean up.

Stack: RAG / source ranking / structured extraction / review notes

Checks: separates fact from assumption / keeps citation trails / exports clean briefs

Team automation

Internal copilot for operations

A copilot pattern for triaging messy requests, retrieving context, preparing drafts, shaping data, and asking for approval before changes land.

Operators move faster while managers and engineers can still inspect what happened and why.

Stack: Next.js / API connectors / RAG / role-aware flows

Checks: intake to next action / draft before send / approval before change

Product build

Full-stack AI product prototype

A real product slice with user flow, model integration, persistence, review states, error paths, and maintainable UI.

Founders get a credible AI product surface that can be tested, demoed, and improved by another engineer.

Stack: Next.js / React / server actions / LLM APIs

Checks: clear user flow / reviewable model output / handoff-ready code

Inspectable outputs

Artifact review system

A table, CSV, XLSX, or structured output layer where AI results carry evidence, owner, blocker, review state, and export readiness.

AI output becomes an operational artifact instead of an unreviewed blob of text.

Stack: TypeScript / CSV / XLSX / schema checks

Checks: row-level evidence / visible blockers / export checks

A boring operating loop for non-boring AI work.

Orchestrate workers only when it helps. Keep dependent work sequential when the next step needs a stable contract. Preserve one clear owner for the final artifact.

01

Start with the business job

Name the user, input, current bottleneck, approval boundary, and output before picking an agent pattern.

02

Design the review surface

Every workflow needs visible state: source, confidence, owner, blocker, next action, and the moment a human must decide.

03

Automate only the safe path

Browser actions, API calls, exports, and customer-visible messages stay gated when the cost of being wrong is high.

04

Ship artifacts, not transcripts

The useful output is a product flow, table, brief, review queue, runbook, or handoff another engineer can maintain.
