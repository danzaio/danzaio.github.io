Scoped contracts, narrow workers, one merge

Orchestrator Worker

Dan turns broad LLM work into a lead agent, specialist worker lanes, evidence rules, and one final owner for the merge.

The lead agent writes the prompt contract, builds the context pack, assigns worker lanes, and keeps one merge point. Workers stay narrow: source lookup, copy, UI, review, or artifact shaping.

Parallel agents do not replace ownership. One operator still reconciles evidence, contradictions, risks, and next action.

Independent lanes with explicit merge criteria

Parallel Processing

Research, implementation, copy review, and review can run side by side when their outputs do not depend on each other.

The safe version is boring: non-goals, public boundaries, acceptance criteria, and a shared merge contract. If one lane needs another lane's artifact, it stays sequential instead.

Do not fan out work that shares a mutable contract unless agents can coordinate around a small, stable interface.

Human in the Loop plus Tool Approval

HIL approval

External effects, exports, risky edits, sensitive data use, and ambiguous tradeoffs pause until the operator can inspect risk.

The approval card names the action, rollback path, public safety boundary, and what happens if approval is denied. It is not a vague confidence check.

If a tool can affect external state or expose sensitive material, approval belongs before the call, not in the closeout.

Agent XLSX Artifact

Rows carry source, decision, owner, blocker, review state, and export readiness so agent output can be audited before downstream use.

The pattern borrows from public AI SDK artifact templates and spreadsheet workflows: rows are treated as work product, not chat prose. The export is public safe and reviewable.

A spreadsheet artifact is only useful if the row schema makes bad, missing, or uncertain data visible.

Execution path

Requests go to a single agent, specialist worker, review lane, approval gate, or sequential workflow based on risk and evidence needs.

The path keeps the system from making every task a swarm. A small prompt copy change, a source-backed research pass, and a multi-page UI implementation should not use the same action space.

The path has to be explainable before execution. If it is unclear, build the context first.

Keep dependent work serial

Sequential workflows

When one step produces the contract another step consumes, Dan keeps the workflow ordered: inspect, scope, build, validate, handoff.

Sequential does not mean slow. It means avoiding waste: the review lane should not start before the surface exists, and the reviewer should not judge a schema that is still moving.

Parallelize independent work, but never parallelize the creation and consumption of the same evolving contract.

Fix bad calls at the source

Tool call repair

Bad arguments, stale paths, malformed output, weak evidence, and missing proof are repaired by changing the call or assumption.

The workflow records what failed and why. That makes repair part of the evidence trail instead of a hidden retry loop that can mask an incorrect source or unsafe action.

Retries are not a strategy when the argument, path, selector, or context pack is wrong.

Artifact export

The output becomes an artifact another operator can inspect: table, CSV, XLSX, plan, code block, proof note, risk list, or next action note.

This is where public component references become workflow demos: source-faithful public component ports, public pattern reconstructions, and compact interaction studies.

Do not export raw captures, sensitive sources, non-public URLs, secrets, or operational details that belong outside public copy.

Context builder

The context pack names current sources, public references, review checks, boundaries, and claims the model may make.

Public component references stay separate from implementation detail: component references assembled into workflow demos, with claims bounded to what the public sources support.

The model does not need sensitive raw material when a public-safe summary and current source list can support the work.

Export-ready artifact discipline.

Work does not end as a chat transcript. It becomes a prompt contract, context pack, table artifact, plan artifact, proof packet, exported spreadsheet, or structured handoff that another operator can inspect.

prompt contract
Role, task, context pack, tool rules, acceptance criteria, refusal path, output shape, risks, and stop condition.
context pack
Current sources, public references, review scope, allowed claims, and excluded sensitive material.
table artifact
Agent XLSX Artifact or CSV handoff with row-level status, source, owner, blocker, unresolved question, and export readiness.
plan artifact
A compact execution plan with worker lanes, allowed tools, approval gates, merge criteria, sequential dependencies, and non-goals.
proof packet
Eval target, tool evidence, review note, review checks, reviewer objection, and final uncertainty.
structured output
Schema-shaped result that can be checked, exported, or handed to the next operator without rewriting.
handoff
Public-safe risks, next action, maintainer note, and what not to carry into memory or downstream artifacts.

Changelog from my journey

I've been working on Aceternity for the past 2 years. Here's a timeline of my journey.

Scope

Scope

Turn the ask into a prompt contract with role, task, context pack, tools, approval points, output shape, risks, and stop rules.

const contract = defineAgent({ role, task, contextPack, tools, approvals, output, risks });

Context

Context

Build the context pack from current sources, public component references, tables, and privacy boundaries.

const context = buildContext({ sources, references, tables, boundary });

Path

Path

Choose a single agent, worker lane, evaluator optimizer, or sequential workflow based on independence and risk.

const path = choosePath({ independence, risk, evidence, outputType });

Orchestrate

Orchestrate

Give each worker a narrow scope, non-goals, evidence rules, and merge contract, then keep one owner for the final decision.

const result = await orchestrate({ planner, workers, reviewer, mergeOwner });

Approve

Approve

Pause for Human in the Loop or Tool Approval before external effects, sensitive data use, artifact export, or risky edits.

await requireApproval({ action, args, risk, rollback, nextStep });

Repair

Repair

When a tool call fails or returns weak evidence, fix the arguments, context, or source assumption instead of hiding the problem.

const repaired = await repairToolCall({ call, error, source, expectedShape });

Artifact

Artifact

Return structured output, table, CSV, plan, code, or Agent XLSX artifacts that can be inspected before the next system consumes them.

const artifact = await createArtifact({ schema, rows, format: 'xlsx', exportReady: true });

Validate

Validate

Check evaluations, surface state, schema validity, approval logs, review notes, and stop conditions.

await validate({ result, surfaceReview, approvalLog, reviewNotes, stopRule });

Handoff

Handoff

Close with public-safe evidence, unresolved risks, the next action, and maintainability notes the next operator can use.

return handoff({ evidence, risks, nextAction, maintainability });
